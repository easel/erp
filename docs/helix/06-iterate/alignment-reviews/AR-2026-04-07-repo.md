## Intent Summary

- **Vision**: Open-source ERP for satellite operators, replacing NetSuite. Seven core domains: financial management, procurement, sales, CRM, orbital asset management, export control, complex delivery logistics, and program management. Isomorphic TypeScript on Bun, self-hosted, local-first.
- **Requirements**: PRD defines 79 requirements across 9 categories (FIN, SCM, SLS, CRM, OAM, EXP, LOG, PGM, PLT). Phase 1 covers P0 requirements: core finance, basic procurement/sales/CRM, export control foundation, basic logistics, and platform infrastructure.
- **Features / Stories**: 9 feature specs (FEAT-001 through FEAT-009) expand PRD requirements into user stories and detailed acceptance criteria. FEAT-009 (Platform) is the most granular with 22 acceptance criteria sections.
- **Architecture / ADRs**: 11 ADRs covering: classification inheritance (ADR-001), UUID v4 keys (ADR-002), NUMERIC(19,6) money (ADR-003), audit log schema (ADR-004), dual customer model (ADR-005), compliance enforcement (ADR-006), fiscal period states (ADR-007), Redis optionality (ADR-008), isomorphic TS/Bun/local-first (ADR-009), frontend validation (ADR-010), ERP component library (ADR-011).
- **Technical Design**: SD-001 (system architecture), SD-002 (data model — 99 tables), SD-003 (Phase 1 implementation plan — 8 work packages), SD-004 (authn provider abstraction).
- **Test Plans**: SD-003 §7 defines unit (90%+ domain logic), integration (real PostgreSQL), E2E (Playwright), compliance test suite, and performance testing strategy.
- **Implementation Plans**: SD-003 defines WP-0 through WP-7. WP-8 (UI) was added as execution work during implementation (not in SD-003).

## Implementation Map

- **Topology**: Bun monorepo with 3 workspace packages: `@apogee/shared` (schemas, types, sync), `@apogee/server` (Fastify + GraphQL Yoga + Pothos, domain services), `@apogee/web` (Next.js 15 + React 19 + Tailwind 4 + shadcn/ui)
- **Entry Points**: Server: `packages/server/src/index.ts` → `app.ts` (Fastify). Web: `packages/web/src/app/layout.tsx` (Next.js App Router). GraphQL at `/graphql`. Health at `/health/live`, `/health/ready`. Metrics at `/metrics`.
- **Database**: PostgreSQL via `graphile-migrate`. 99 tables in `current.sql` (2595 lines). Covers: platform (legal_entity, user_account, RBAC, audit_entry), finance (COA, GL, AP, AR, intercompany, elimination), compliance (screening, holds, country restrictions, restricted regions), procurement (vendors, POs, goods receipt, inventory), sales (customers, products, quotes, orders, returns), CRM (companies, contacts, opportunities, activities, leads, pipeline stages), logistics (carriers, shipments, customs docs), auth (sessions, identity links).
- **Domain Services**: Finance (7 files, 2614 lines), Compliance (1 file, 482 lines), Procurement (3 files, 638 lines), Sales (3 files, 931 lines), Logistics (2 files, 882 lines), Auth (6 files, 674 lines).
- **GraphQL**: 1335 lines. 29 query fields (list + detail for all entity types). 5 mutations (createVendor, createJournalEntry, submitPO, approvePO, sendPO). Pagination via PaginationInput. Entity-scoped queries (16 WHERE entity_id clauses).
- **Zod Schemas**: 16 entity schema files in `packages/shared/src/entity-schemas/`.
- **Frontend**: Next.js App Router with 5 module layouts (finance, sales, procurement, crm, compliance). 28 page files. 14 components + 9 shadcn/ui components. Tailwind CSS 4 installed and configured. React Hook Form + Zod validation on vendor create and journal entry forms.
- **Tests**: 976 tests across 54 files (server: 35, shared: 7, web: 5, e2e: 7). Test categories: unit (domain logic), integration (DB services), E2E (Playwright browser + API), security audit, performance benchmarks. 14 skip, 7 fail, 7 errors in last run.
- **CI**: GitHub Actions with Bun primary + Node.js fallback jobs. Lint, typecheck, test.
- **Infrastructure**: Helm chart at `deploy/helm/apogee/` with dev/prod values. Kind cluster demo via `scripts/demo.ts`. Docker Compose for local PostgreSQL + Redis.
- **Documentation Site**: Hugo + Hextra at `website/` with entity catalog.
- **Unplanned Areas**: WP-8 (UI wiring) was not in SD-003. Hugo site was added without a governing spec update.
