## Intent Summary

- **Vision**: Open-source ERP for international satellite operators, functionally equivalent to NetSuite, with native orbital asset management, export control, complex delivery logistics, and multi-entity consolidation.
- **Requirements**: PRD defines 7 goals (G1–G7) and ~100 requirements across 9 domains (FIN, SCM, SLS, CRM, OAM, EXP, LOG, PGM, PLT), prioritized P0–P3 across 4 phases.
- **Features / Stories**: 9 feature specifications (FEAT-001 through FEAT-009) with full traceability to PRD requirements and explicit acceptance criteria per requirement.
- **Architecture / ADRs**: Layered modular monolith (SD-001). 11 ADRs covering key decisions: UUID PKs, NUMERIC(19,6) money, immutable audit, dual-entity customers, fiscal period state machine, Redis optionality, isomorphic TS+Bun, frontend validation, component library.
- **Technical Design**: SD-001 (system architecture), SD-002 (data model, 50+ entities), SD-003 (Phase 1 implementation plan, 8 work packages), SD-004 (auth provider abstraction).
- **Test Plans**: No standalone test plan documents. Tests are co-located with implementation. 41 test files, 858 passing tests, 1499 expect() calls.
- **Implementation Plans**: SD-003 defines WP-0 through WP-7 with dependencies, acceptance criteria, and definitions of done.

## Implementation Map

- **Topology**: Bun monorepo with 3 packages: `@apogee/shared` (types/schemas), `@apogee/server` (Fastify API + domain services), `@apogee/web` (React/Next.js components)
- **Entry Points**: `packages/server/src/index.ts` (buildApp factory + conditional listen), `packages/web/src/index.ts` (component barrel export), `packages/shared/src/index.ts` (schema barrel export)
- **Public Interfaces**: GraphQL endpoint at `/graphql` (Pothos schema), REST health/metrics endpoints, auth routes (`/auth/oidc/*`, `/auth/saml/*`, `/auth/mfa/*`, `/api/v1/auth/sessions`)
- **Test Surfaces**: 41 test files across 3 packages; 858 tests passing; server: 30 files (auth, finance, procurement, logistics, e2e, security, perf); shared: 7 files; web: 5 files
- **Feature Flags**: Redis optional (ADR-008); NODE_ENV controls logging/GraphiQL; auth provider pluggable (OIDC/SAML/stub)
- **Build/Deploy**: Multi-stage Dockerfile, Helm chart with dev/prod values, GitHub Actions CI (Bun + Node.js dual runtime)
- **Unplanned Areas**: `rpt-service.ts` duplicates `reporting-service.ts` with slightly different APIs
- **Dead/Orphaned**: `auth/routes/defaults.ts` referenced in exploration but does not exist (no impact)
